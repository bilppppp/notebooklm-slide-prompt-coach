# 中文风格索引

这份索引是给中文使用场景准备的轻量参考表。

用法：

- 当用户不熟悉英文风格名时，优先用这里的中文名称沟通
- 当用户只会说“我想要那种高级感 / 媒体感 / 科技感 / 手绘感”时，用这里做轻量映射
- 真正拼最终 YAML 提示词时，再结合 `style-map.md`、`style-catalog-extended.md` 和 `output-template.md`

注意：

- 这份索引不是拿来给用户做题的
- 不要让用户在这里逐项打勾
- 它只是帮助你把用户已经喜欢的感觉说清楚

## 1. 内容讲解型

### 黑板讲解风

适合：

- 培训
- 教学
- 逐步解释
- 概念入门

关键词：

- 粉笔感
- 手写感
- 课堂感
- 推导过程

不适合：

- 高端品牌汇报
- 很强的商务气场

### 白板整理风

适合：

- 内部分享
- 头脑风暴
- 工作坊总结

关键词：

- 马克笔
- 即时记录
- 清爽
- 协作感

### 可视化笔记风

适合：

- 会议纪要
- 工作坊
- 组织文化
- 以人为中心的内容

关键词：

- 手绘图标
- 箭头连接
- 亲和
- 现场感

## 7. 轻量对照口诀

如果用户只想快点定，可以这样轻轻对照：

- 要讲清楚：选内容讲解型
- 要讲结构：选结构与技术型
- 要讲战略：选商务与高端型
- 要讲人和团队：选人物与亲和型
- 要抓眼球：选编辑、杂志与潮流型或流行、娱乐与高冲击型

## 8. 推荐表达方式

和用户沟通时，优先用这种说法：

- “偏高端极简商业风”
- “偏商业媒体风”
- “偏蓝图工程风”
- “偏可视化笔记风”
- “偏杂志封面风”

不要一上来只丢英文名，也不要像在做风格测试。

可以用中文名 + 英文参考名的方式补充：

- 高端极简商业风（接近 sharp-edged minimalism）
- 商业媒体风（接近 modern newspaper）
- 杂志封面风（接近 magazine style）

## 9. 转成 YAML 时要保留什么

不管选哪种风格，转成最终 YAML 时都尽量保留这几类信息：

- 目标
- 受众
- 风格方向
- 参考风格家族
- 配色
- 版式节奏
- 图像语言
- 需要避免的东西
